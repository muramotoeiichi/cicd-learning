wanttodo.md
個人の学習用であれば、まずは**「自分の書いたコードが、クラウド上で自動的に入れ替わる快感」**を最短距離で味わうのが一番です。

エンジニアとしての長年のご経験を活かしつつ、モダンな開発環境（AIエージェント等）との相性も良い、以下の構成を具体的におすすめします。

### おすすめの組み合わせ
* **フレームワーク**: **FastAPI** (Python)
    * 型定義がしっかりしており、Swagger UI（APIドキュメント）が自動生成されるため、結合テストの結果をブラウザでも確認しやすいです。
* **デプロイ先**: **Render** (Freeプラン)
    * 「Deploy Hook」というURLを叩くだけでデプロイできる仕組みがあり、GitHub Actionsとの連携が非常にシンプルです。

---

### ステップ・バイ・ステップの構成案

#### 1. 最小限のAPIと「結合テスト」
まずは、実際にDBや外部APIに接続した体（てい）で動かすテストを書きます。

* **App**: `/items/{id}` にアクセスするとデータを返す。
* **Test**: `TestClient` を使い、実際にURLを叩いて `200 OK` と正しいJSONが返るかを検証する。

#### 2. GitHub Actions で「わざと失敗」させる
`.github/workflows/main.yml` を作成し、まずは正常に動かします。その後、**わざとAPIのレスポンス形式を変えてプッシュ**してみてください。

* GitHubの「Actions」タブで赤いバツ印（失敗）がつく。
* デプロイ工程がスキップされる。
* **「テストが守ってくれた」という実感**が得られます。

#### 3. 「デプロイ完了」をブラウザで確認
テストを修正してプッシュし、Actionsが緑色（成功）になったら、RenderのURLを叩きます。自分の手元のコード変更が、数分後に世界中に公開される流れが完結します。

---

### さらに一歩進んだ「実感」を得るなら
最近触られている **Ollama** や **MCP (Model Control Protocol)** などのローカルLLM環境と組み合わせるのも面白いかもしれません。

> **例えば：**
> 「特定のキーワードを投げると、ローカルのOllamaが要約して返すAPI」を作り、その精度や応答速度をCIでチェックする…といった構成にすると、最新技術の実験場としても楽しめます。

まずは「FastAPI + GitHub Actions + Render」の最小構成で、Hello Worldをデプロイしてみませんか？もしよろしければ、具体的なディレクトリ構成や、最初の `yml` ファイルの書き方をお出ししましょうか？
